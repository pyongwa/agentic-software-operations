# Three-surface simultaneous post

The same report lands on three surfaces, in this order, **one comment / one
message per period**:

1. **Tracker** (the Epic / Idea comment) — the **system of record**. The
   authoritative status lives here.
2. **Knowledge base** (a weekly page) — the durable, human-readable narrative;
   accumulates week over week.
3. **Chat** (the channel) — **supporting evidence** and visibility; the
   collaboration-visible trail.

## Rules
- **One comment / one message per period** — never merge periods; post oldest to
  newest if backfilling.
- **Tracker = system of record; chat = supporting evidence.** When they disagree,
  the tracker wins.
- Use the same core wording across surfaces (chat may be shorter).
- Preserve live links — readable titles linked to URLs; keys as traceability.
- Config-driven targets (KB space, channel) from `productops.toml`; never a
  hard-coded channel ID.
